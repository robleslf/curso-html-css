Variables globales

    ---HTML
    <div class="tool-box">
      <ul>
      <li id="rojo-grande">rojo-grande</li>
      <li class="verde-mediano">verde-mediano</li>
      <li att="azul-pequeno">azul-pequeno</li>
      <li>chocolate</li>
      </ul>
    </div>

    ---CSS
    :root {
    --margen: 100px;
    }

    
    ul {
      margin-left: var(--margen);
    }

Variables definidas en contexto definido

    ---HTML
    <div class="tool-box">
      <ul>
      <li id="rojo-grande">rojo-grande</li>
      <li class="verde-mediano">verde-mediano</li>
      <li att="azul-pequeno">azul-pequeno</li>
      <li>chocolate</li>
      </ul>
    </div>

    -CSS
    /*Varibles en contexto definido*/
    .tool-box {
      --rojo: rgb(255, 0, 0);
      --verde: rgb(0, 255, 0);
      --azul: rgb(0, 0, 255);
      --chocolate: rgb(121, 60, 17);
    
      --grande: 40px;
      --mediano: 20px;
      --pequeno: 10px;
    }
    
    /*Selectores*/
    #rojo-grande{
      color: var(--rojo, rgb(238, 130, 238));
      font-size: var(--grande);
    }
    
    .verde-mediano{
      color: var(--verde, fuchsia);
      font-size: var(--mediano);
    }
    
    [att="azul-pequeno"]{
      color: var(--azul, magenta);
      font-size: var(--pequeno);
    }
    
    li {
      color: var(--chocolate, pink);
    }

    }

Compatibilidad entre navegadores

Si un navegador, como p.e. Google Chrome, es capaz de interpretar variables CSS, entonces, suponiendo que establezcamos una varible para el color, podría ser utilizada así:

  ---CSS
  .paleta-color {
    --rojo: rgb(255,0 ,0);
  }

  .texto-rojo {
    color: var(--rojo, color_de_respaldo)
  }

En el ejemplo anterior, si por algún motivo (escribir mal el nombre de la variable --roja ), se utilizaría el color_de_respaldo.

Cuando un navegador no interpreta variables CSS, la solución será incluir la misma propiedad antes de la propiedad que usa la variable.

  ---CSS
  .paleta-color {
    --rojo: rgb(255,0 ,0);
    --verde: rgb(0, 255, 0)
  }

  /*CASO A - Sin color de respaldo*/
  .texto-rojo {
    color: rgb(255, 0, 0);
    color: var(--roja)
  }
  
  /*CASO B - Con color de respaldo*/
  .texto-verder {
    color: rgb(0, 255, 0);
    color: var(--verde, green)
  }

Es necesario seguir utilizando el color_de_respaldo ya que si se omite y has escrito mal el nombre de la variable, no se mostrará color alguno (CASO A), ya que:

  1. El navegador leerá e interpretará la propiedad: 
    color: rgb(255, 0, 0);
  2. Se interpretará la propiedad: 
    color: var(--roja)

    Pero ya que la función var() en la propiedad ha sido interpretada pero el nombre de la variable ha sido mal escrito y no existe un color de respaldo, y además CSS aplicará la última propiedad válida, no se mostrará adecuadamente en el navegador.
